# 扔鸡蛋问题总结

> 问题:$N$ 层楼,某层及以上扔鸡蛋必碎、以下必不碎,要**确定**这个临界层。手里有 $k$ 个鸡蛋(碎了就少一个),问最优策略在**最坏情况**下最少需要扔几次。代码见 [throw_egg.py](throw_egg.py)。

本文分析它**是什么类型**的问题、两种解法**思路的本质区别**、以及背后**可迁移的宏观框架**。

---

## 1. 它是什么:无环 + min-max 最坏情况 DP

扔鸡蛋是**动态规划**,但要和 MDP 区分开两个关键性质:

**(a) 状态不可回溯 —— 状态图是 DAG(有向无环图)。**
子问题严格变小(楼层数 $\downarrow$、鸡蛋数 $\downarrow$),永远不会绕回自己。因此**按拓扑序扫一遍就能精确解出**,不需要像 MDP 那样迭代到收敛。
- 对比 MDP:状态可以绕回自己($v(s)$ 依赖 $v(s')$,而 $v(s')$ 又可能依赖回 $v(s)$),有环、无拓扑序,**才必须反复迭代**。

**(b) 聚合后继用 `max`(最坏情况),不是期望。**
扔一次后分裂成"碎 / 不碎"两个后继,取 $\max$ 表示"保证最坏情况也能搞定";标准 MDP 用的是**期望** $\sum_{s'}P\cdot v(s')$。
- 所以扔鸡蛋是 **minimax / 最坏情况 DP**,不是期望意义的随机 MDP。

| | 扔鸡蛋 | MDP(value/policy iteration) |
|---|---|---|
| 状态图 | 无环 DAG,不可回溯 | 有环,可回溯 |
| 求解 | 一遍 backward induction(倒推) | 迭代到收敛 |
| 后继聚合 | `max`(最坏情况) | 期望 $\sum P\cdot v$ |
| 目标 | 保证最坏 | 优化期望 |

---

## 2. 两种解法

### 方法1(正向):楼层 → 次数

状态 `d[n][k]` = $n$ 层、$k$ 蛋时最少次数。枚举第一颗扔第 $i$ 层:

$$
d[n][k] = \min_{i}\Big(\max(\underbrace{d[i-1][k-1]}_{\text{碎了:下方}},\ \underbrace{d[n-i][k]}_{\text{没碎:上方}}) + 1\Big)
$$

- 时间 $O(K N^2)$,空间 $O(KN)$。内层枚举决策 $i$ 是 $O(N^2)$ 的来源。

### 方法2(反向):次数 → 楼层

状态 `d[k][m]` = $k$ 蛋、$m$ 次能**确定的最多楼层数**。第一扔分裂为碎/不碎两段,两段相加:

$$
d[k][m] = \underbrace{d[k-1][m-1]}_{\text{碎了,下方}} + \underbrace{d[k][m-1]}_{\text{没碎,上方}} + 1
$$

从小到大递增 $m$,直到 $d[K][m] \ge N$。

- 时间 $O(K N^{1/K})$,空间同阶(滚动数组可降到 $O(K)$)。
- 推导:$d[K][m] \sim m^K/K! \ge N \Rightarrow m = O(N^{1/K})$。$K=2$ 时为 $O(\sqrt N)$:$N=100 \Rightarrow m=14$,比方法1的 $\sim 10^4$ 快几个数量级。

**注意方法2里没有 `min/max`** —— 优化被消掉了(原因见第 4 节)。

---

## 3. policy 提取:最优 value 唯一,最优 policy 不唯一

两种解法都能 extract 出"第一颗蛋该扔哪层"的策略:

- **方法1**:policy 是 $\arg\min_i$,需额外用 `choice[n][k]` 记录取到最优的那个 $i$。
- **方法2**:没有显式 $\arg\min$,policy 藏在加法 split 里——起扔层 $= d[k-1][m-1]+1$。

两者算出的最优**次数(value)必然相同**(都是 14),但 extract 出的 **policy 可以不同**:

| | 第一扔起始层 | 间隔序列 |
|---|---|---|
| 方法1(`if cost < ...` 取最小最优 $i$) | 9 | 9,13,12,…,2,1 |
| 方法2(构造法,平衡解) | 14 | 14,13,12,…,4,1 |

状态 `(100层, 2蛋)` 下第一扔在 9~14 层都能保证最坏 14 次,是个**有多个最优解**的状态。

> 这正是 MDP 的通性:**最优 value 唯一,但并列最优的动作可以有多个,最终 policy 取决于 $\arg\max/\arg\min$ 的 tie-breaking。**

---

## 4. 思路本质(问题的结构/特性):目标与约束对调 + 单调前沿可逆

方法1 和方法2 是**同一问题的两种 indexing**,把它写成带约束的优化就看清了:

| | 方法1(正向) | 方法2(反向) |
|---|---|---|
| 形式 | $\min\ \text{trials}\quad\text{s.t. floors}\ge N$ | $\max\ \text{floors}\quad\text{s.t. trials}\le m$ |
| 一句话 | 要覆盖 $N$ 层,最少花几次 | 给我 $m$ 次,最多覆盖几层 |

这就是**目标与约束对调 (objective ↔ constraint swap)**。两者靠**单调性**连起来:记 $f(m)=$「$m$ 次能覆盖的最多楼层」,它随 $m$ 单调递增,于是

$$
\text{原问题答案} = f^{-1}(N) := \min\{\,m : f(m)\ge N\,\}
$$

solution2 本质就是**对一个单调函数求(广义)逆**(和"二分答案"同源)。

> **精确化**:这里的 $f^{-1}$ 是**广义逆 / 阈值逆**,不是双射意义的逆函数。
> - 真逆函数要求 $f$ **严格**单调(单射);**弱单调**(有水平段)时 $f$ 不是单射,没有真逆。
> - 但阈值逆 $\min\{m:f(m)\ge N\}$ 只要 $f$ **单调(弱即可)** 就良定义,且能容忍两种真逆活不下来的情况:① $f$ 有水平段;② $N$ 不在 $f$ 的值域里(整数值、跳变)。
> - 扔鸡蛋的 $f$ 恰好是**整数值、跳跃**的,大多数 $N$ 不在其像里,但阈值逆始终存在。
> - 所以"二分答案"普适的根因是**单调性使阈值集上闭 (upward-closed)**,而非双射可逆。

**为什么对调能消掉 `min`:**
- 方法1 在"楼层→次数"下,扔第几层 $i$ 是**自由变量**,必须 $\min_i$ 枚举。
- 方法2 在"次数→楼层"下要让覆盖**最大化**,最优第一扔被**结构唯一逼出**:碎了的下方必须正好填满 $d[k-1][m-1]$ 层、没碎的上方正好填满 $d[k][m-1]$ 层,**没有 slack 也就没得选** —— 优化塌缩成闭式递推。

**它和优化里的"对偶"是不是一回事?** —— 形似神不同:

| | solution2 的互换 | Lagrangian / LP 对偶 |
|---|---|---|
| 产物 | 同一问题换索引,值**恒等** | 一个**新的**优化(在乘子上) |
| 新变量 | 无 | 对偶变量/影子价格 $\lambda$ |
| 值关系 | 恒等(同一函数的 $f$ 与 $f^{-1}$) | 弱对偶 $d^*\le p^*$,强对偶需凸性 |
| 对偶间隙 | 无此概念 | 可能有 |

共通的只有"**把谁当目标、把谁当约束**"这层角色对调的直觉;真对偶在此之上还套了乘子 + min-max + 间隙那套机器,solution2 没用到。更准确的名字是 **inverse / parametric reformulation**,而非 dual。

## 5. 宏观框架:最优值函数是一条权衡前沿

> 任何带约束的最优化背后都有**同一条单调曲线(Pareto 前沿)**:横轴投入的资源,纵轴能达到的成就。`min cost s.t. benefit≥b` 和 `max benefit s.t. cost≤c` 不是两个问题,而是**同一条曲线的两种读法**。
>
> **解一个最优化 = 描这条曲线,而你能挑从哪个轴去描。**

描同一条曲线的三种方式:

| 方式 | 怎么探 | 对应 |
|---|---|---|
| 沿成就轴 | 固定 benefit,求最小 cost | 方法1 |
| 沿资源轴 | 固定 cost 预算,求最大 benefit | 方法2 |
| 沿斜率 | 给约束价格 $\lambda$,看支撑超平面 | Lagrangian / Aliens trick(WQS 二分) |

### 三个可操作的"怎么想到"触发器

1. **对调维度**:DP 是 `状态→值`。若**值域远小于某个状态维度**,把值挪去当索引。(扔鸡蛋:次数 $\sim\sqrt N \ll N$ 层)
2. **求逆**:想要的若是某个**易正向算的单调量的逆**,就正向算再二分/反查。(二分答案)
3. **消元**:重标状态时刻意让最优子结构**无 slack**,$\min_a$ 就塌掉。

### 跨领域同构(记一个等于记一片)

- **0/1 背包**:`dp[i][w]=最大价值` ↔ `dp[i][v]=最小重量`,价值范围小于容量时换后者。**与扔鸡蛋完全同构**。
- **LIS 的 $O(n\log n)$**:朴素 `dp[i]=最长长度`;快版**按长度索引、存该长度的最小结尾值** —— 正是触发器1的对调。
- **二分答案 / 参数搜索**:触发器2 的纯粹形态。

### 连回 RL

RL 的 **value function $v_*(s)$ 本身就是这种前沿对象**(状态 → 最优长期成就)。"最优 value 唯一、policy 不唯一",在曲线图景里就是:**曲线唯一,但达到某点的走法可以多条**。value iteration 的一次次 backup,就是在反复逼近这条曲线。

> **边界**:这套只在**单调性成立**时有效。非单调、或换轴后决策仍有真实自由度时,对调救不了,该枚举还得枚举。它是强启发,不是万能公式。

## 6. 与 value/policy iteration 的关系(避免误套)

容易把"两种解法"误当成"value iteration vs policy iteration",但它们在**两条正交的轴**上:

- **轴 A(怎么解 Bellman 系统)**:VI ↔ PI。扔鸡蛋是 DAG,这个轴"塌缩"——一遍扫完,VI/PI 无差别,PI 失去意义。
- **轴 B(状态怎么参数化)**:方法1 ↔ 方法2。这是换元,和轴 A 无关。

| | 对应到 RL |
|---|---|
| 方法1 | Bellman **最优方程** + 一遍 backward induction(VI 在 DAG 上的退化);policy 来自 argmin |
| 方法2 | 同一最优值的**对偶/换元**解法,消去了动作优化,无 VI/PI 对应物 |
| Policy Iteration | 在扔鸡蛋里**无真正对应**——DAG 上评估也是一遍扫完,不存在要迭代的 evaluation 不动点 |
